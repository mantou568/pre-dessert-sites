# Media-Saber 请求字段完整列表

## 1. user_basic_info（用户基础信息）

| 字段名 | 中文注释 | 说明 |
|--------|---------|------|
| `is_login` | 是否登录 | 判断用户是否已登录 |
| `signed_in` | 是否签到 | 判断用户今日是否已签到 |
| `id` | 用户ID | 用户的唯一标识符 |
| `name` | 用户名称 | 用户的显示名称/用户名 |
| `ratio` | 分享率 | 上传量/下载量的比值 |
| `uploaded` | 上传量 | 用户总上传流量（字节） |
| `downloaded` | 下载量 | 用户总下载流量（字节） |
| `bonus` | 魔力值 | 站点的魔力/积分点数 |
| `gold` | 金币 | 特殊站点使用的金币，最终转换为魔力 |
| `silver` | 银币 | 特殊站点使用的银币，最终转换为魔力 |
| `copper` | 铜币 | 特殊站点使用的铜币，最终转换为魔力 |

## 2. user_details（用户详情信息）

| 字段名 | 中文注释 | 说明 |

| 字段名 | 中文注释 | 说明 |

## 字段数据类型说明

所有字段的数据类型支持字段转换，目前支持：
- `string` → `int`
- `string` → `int64`
- `string` → `bool`
- `string` → `float64`
- `string` → `string`

如需更多类型请在 `client.decodeHookFunc` 代码里添加。

## 注意事项

1. **时间字段**：所有时间相关字段（`join_at`、`last_accessed`、`date_added`、`date` 等）应使用 `timestamp` 过滤器转换为 Unix 时间戳
2. **体积字段**：所有体积相关字段（`size`、`uploaded`、`downloaded` 等）应使用 `byte_size` 过滤器转换为字节数
3. **相对地址**：`details`、`download`、`link` 等字段应使用相对地址，系统会自动拼接域名
4. **可选字段**：某些字段可能在某些站点不存在，可以使用 `any` 字段按顺序尝试多个选择器
